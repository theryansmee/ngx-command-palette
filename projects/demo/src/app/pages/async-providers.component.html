Async Search Providers

Register API-backed search sources that return results asynchronously.

Basic Provider

A provider without a prefix fires on every query:

this.palette.registerProvider(
  {
    id: 'doc-search',
    category: 'Documentation',
    minQueryLength: 2,
    debounce: 300,
    search: (query) => this.http.get<Doc[]>(`/api/docs?q=${query}`).pipe(
      map(docs => docs.map(doc => ({
        id: `doc:${doc.id}`,
        label: doc.title,
        action: () => window.open(doc.url),
      }))),
    ),
  },
  this.destroyRef,
);

Prefixed Provider

A provider with a prefix only fires when the user types that prefix. The prefix is stripped before being passed to the search function. 

this.palette.registerProvider(
  {
    id: 'user-search',
    category: 'Users',
    prefix: '@',
    minQueryLength: 2,
    debounce: 300,
    search: (query) => this.userService.search(query).pipe(
      map(users => users.map(user => ({
        id: `user:${user.id}`,
        label: user.name,
        action: () => this.router.navigate(['/users', user.id]),
      }))),
    ),
  },
  this.destroyRef,
);

With this setup, typing @john only hits the user API. Typing dashboard only searches static commands. Registered prefixes are automatically shown as hints in the palette footer.

Provider Interface

Property Type Description
id string Unique identifier
category string Group heading for results
search (query: string) => Observable<Command[]> The search function
prefix string? Prefix trigger (e.g. @, #)
debounce number? Debounce in ms (default: 300)
minQueryLength number? Minimum chars before searching (default: 1)

Loading State

The palette shows a "Searching..." indicator while providers are in-flight. You can also read the state programmatically:

const isLoading: boolean = this.palette.loading();
