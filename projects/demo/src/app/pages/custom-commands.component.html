Custom Commands

Register your own actions alongside auto-registered routes.

Registering Commands

Inject CommandPaletteService and call register(). Pass a DestroyRef to auto-deregister when the component is destroyed.

import { Component, inject, DestroyRef } from '@angular/core';
import { CommandPaletteService } from '@theryansmee/ngx-command-palette';

@Component({ ... })
export class ProjectListComponent {
  readonly #palette = inject(CommandPaletteService);
  readonly #destroyRef = inject(DestroyRef);

  constructor() {
    this.#palette.register([
      {
        id: 'create-project',
        label: 'Create New Project',
        category: 'Actions',
        shortcut: 'Cmd+N',
        keywords: ['new', 'add'],
        priority: 10,
        action: () => this.openCreateDialog(),
      },
    ], this.#destroyRef);
  }
}

Command Interface

Property Type Description
id string Unique identifier
label string Display text
category string? Group heading (e.g. "Pages", "Actions")
icon string? Icon name or identifier
keywords string[]? Additional search terms
shortcut string? Display-only shortcut hint
action () => void What happens when executed
priority number? Ranking boost (higher = appears first)
context CommandContext? Visibility constraints

Contextual Commands

Commands can be scoped to specific routes or dynamic conditions using the context property.

this.palette.register([
  {
    id: 'delete-project',
    label: 'Delete Project',
    action: () => this.deleteProject(),
    context: {
      routes: ['/projects/*'],
      when: () => this.canDelete(),
    },
  },
], this.destroyRef);
  • routes accepts glob patterns. * matches a single segment, ** matches any depth.
  • when is a function returning a boolean, re-evaluated on each search.
  • If both are provided, both must pass for the command to be visible.
  • Commands without a context are always visible.
