Custom Item Templates

Override how result rows are rendered, globally or per-category. Available from v22.1.0.

By default, each result row renders an icon, label, and shortcut badge. For richer results you can provide custom ng-template elements inside <cmd-palette> that override row content while keeping keyboard navigation, accessibility, and active-state behavior intact.

Importing the Directive

Import CmdItemTemplateDirective in the component where you use <cmd-palette>: 

import { CmdPaletteComponent, CmdItemTemplateDirective } from '@theryansmee/ngx-command-palette';

@Component({
  imports: [CmdPaletteComponent, CmdItemTemplateDirective],
  ...
})
export class AppComponent {}

Global Template

A template without a category value applies to all result items:

<cmd-palette>
  <ng-template cmdItemTemplate let-command let-active="active">
    <img [src]="command.data?.['avatar']" class="avatar" />
    <div class="details">
      <span>{{ command.label }}</span>
      <span class="description">{{ command.data?.['description'] }}</span>
    </div>
  </ng-template>
</cmd-palette>

Per-Category Template

Provide a category name to scope the template to a specific group. Items in other categories fall back to the next matching template or the built-in default: 

<cmd-palette>
  <ng-template cmdItemTemplate="Jira Issues" let-command let-active="active">
    <span class="status-badge" [class]="command.data?.['status']">
      {{ command.data?.['status'] }}
    </span>
    <span>{{ command.label }}</span>
    <span class="ticket-id">{{ command.data?.['ticketId'] }}</span>
  </ng-template>

  <ng-template cmdItemTemplate="Team Members" let-command let-active="active">
    <img [src]="command.data?.['avatar']" class="avatar" />
    <span>{{ command.label }}</span>
    <span class="role">{{ command.data?.['role'] }}</span>
  </ng-template>
</cmd-palette>

Resolution Order

When rendering an item, the palette resolves its template in this order:

  1. Category-specific template matching the item's category
  2. Global template (one with no category value)
  3. Built-in default (icon + label + shortcut)

This means you can mix and match: provide a per-category template for your search provider results while letting routes and custom commands use the default rendering.

Template Context

Variable Type Access Description
$implicit Command let-command The command object for this row
active boolean let-active="active" Whether this row is the currently selected item

The CmdItemTemplateContext interface is exported for type-safe usage in your components.

Using the data Property

The data property on Command is an optional Record<string, unknown> for attaching arbitrary metadata that your custom templates can render. Search providers are the primary use case: 

this.palette.registerProvider(
  {
    id: 'ticket-search',
    category: 'Jira Issues',
    search: (query) => this.ticketService.search(query).pipe(
      map(tickets => tickets.map(ticket => ({
        id: `ticket:${ticket.id}`,
        label: ticket.title,
        action: () => this.router.navigate(['/tickets', ticket.id]),
        data: {
          ticketId: ticket.key,
          status: ticket.status,
          assignee: ticket.assignee,
        },
      }))),
    ),
  },
  this.destroyRef,
);

Design Notes

  • Custom templates only control the inner content of each row. The host element still handles role="option", aria-selected, active state, click handling, and scroll-into-view behavior.
  • Keyboard navigation and accessibility work exactly the same regardless of which template is rendered.
  • The active context variable lets you add visual indicators (e.g. a highlight or checkmark) to the active row if needed.
