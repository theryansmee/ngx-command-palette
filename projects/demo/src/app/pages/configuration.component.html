Configuration

Customise the palette behaviour via the provider config.

provideCommandPalette({
  shortcut: 'mod.k',
  placeholder: 'Search...',
  autoRegisterRoutes: true,
  maxResults: 10,
  recentCount: 5,
  debounce: 150,
})

Options

Option Type Default Description
shortcut string 'mod.k' Keyboard shortcut to open the palette
placeholder string 'Search or type a command...' Input placeholder text
autoRegisterRoutes boolean true Auto-register routes from Router config
maxResults number 10 Maximum results shown
recentCount number 5 Number of recent commands tracked
debounce number 0 Input debounce in milliseconds

Shortcut Format

The shortcut string uses dot-separated modifier keys followed by the key.

Shortcut Keys
mod.k Cmd+K (Mac) / Ctrl+K (Windows/Linux)
meta.k Cmd+K (Mac) / Win+K (Windows)
ctrl.k Ctrl+K
ctrl.shift.p Ctrl+Shift+P
meta.shift.p Cmd+Shift+P

Route Configuration

Routes with a title property are auto-registered. You can enrich them with extra metadata:

{
  path: 'settings/billing',
  component: BillingComponent,
  title: 'Billing',
  data: {
    commandPalette: {
      label: 'Billing & Payments',
      category: 'Settings',
      keywords: ['invoice', 'payment', 'subscription'],
      priority: 5,
    },
  },
}

Set commandPalette: false in route data to exclude a route. Parameterised routes (e.g. :id) are excluded automatically unless you provide an explicit config.
